3.3  研究方法
3.3.1  智能体框架
在研究过程中，我们将重点放在了如何搭建有足够效果且鲁棒的智能体系统。在这一过程中，我们最开始采用MetaGPT框架进行多智能体系统的构建，但是在构建过程中，我们发现庭审过程中多智能体的交互较为复杂，难以构建法庭辩论环节的辩论流程。经过多个智能体代码框架的测试，我们最终选择了AgentScope作为多智能体系统的代码框架。AgentScope是由阿里开源的多智能体开发框架，具有全面的试错报错机制和完善的应用流程编排，同时具备高易用和高鲁棒的特性。我们使用AgentScope中的message hub类搭建多智能体系统的交互环境，并利用host身份在环境中向多智能体指示庭审流程的进展，防止因模型上下文问题造成的遗忘。

3.3.2  prompt工程
在标准流程构建完成后，我们通过案例分析的方法初步探索当前系统可能存在的问题。在实验的过程中，我们发现了智能体在流程推进中会出现遗忘自己的身份，生成格式错误的回答，对错误的对象进行提问等情况。这些情况表现出当前系统在指令遵循能力上的缺陷，因此我们通过prompt工程对其进行优化。在最开始，我们采用手动的调整方式，使用更加合理的prompt结构对生成内容进行形式化，如“身份信息+职责预设+输出样例”，初步缓解了智能体回答出现错误的现象。同时，我们也尝试了SPO(Self-Supervised Prompt Optimization)方法对prompt工程进行优化，SPO是一种基于大语言模型自监督能力的提示优化框架，与传统依赖人工标注或基准答案的方法不同，SPO通过对比不同提示生成的输出质量，自主完成优化迭代。

3.3.3  法律增强
在之后的实验中，我们发现通用模型缺乏充足的法律知识，并且由于模拟法庭的任务为zero-shot，模型容易对案件作出不合理的罪名预测和刑期判断。针对这一问题，我们采用了模型微调和RAG技术提升多智能体系统生成判决的质量。我们尝试了多种开源和闭源的通用模型和法律微调模型，最终选定了通义法睿模型作为法律相关智能体的基座模型。RAG技术我们利用了现有的开源数据集，如CAIL，LeCaRD等，通过数据集中的案件构建本地的知识库。

3.3.4  RAG
在法律增强阶段，我们引入了检索增强生成（Retrieval-Augmented Generation，简称RAG）技术，以提升多智能体系统在生成判决时的质量。RAG通过将外部知识库与生成模型相结合，提供更准确且上下文相关的回答，减少模型产生幻觉的可能性。 我们利用现有的开源法律数据集，构建本地知识库。这些数据集包含大量的法律条文、案例和判决文书，为系统提供丰富的法律知识来源。将知识库中的法律文档转化为向量表示，以便于高效检索。我们采用先进的acge_text_embedding嵌入技术，将文本内容编码为向量，确保相似内容在向量空间中距离更近。当智能体需要生成法律相关的回答时，首先根据输入查询从向量化的知识库中检索相关文档。这确保了模型能够获取最新且相关的法律信息，增强回答的准确性。将检索到的相关法律文档与用户的查询一起输入到生成模型中，生成上下文相关且准确的回答。

案例背景	原告张某是一家互联网公司的前员工，在该公司连续工作了三年，近期被公司辞退后，公司只向其支付了一个月工资的经济补偿金。张某认为公司做法违法，向法院提起诉讼，请求公司支付剩余两个月的经济补偿金。
不使用 RAG 技术的模拟庭审陈述	被告方律师（LLM纯生成陈述）：
尊敬的审判长，我们认为本案中公司向原告支付了一个月经济补偿金，并未违反相关法律规定。根据我国《劳动合同法》，用人单位解除劳动合同的，应当根据劳动者的工作年限支付经济补偿。本案中原告工作了三年，公司支付了补偿金，已经履行了法律义务。因此，我们请求法院驳回原告的诉讼请求。
使用 RAG 技术后的模拟庭审陈述	被告方律师（RAG检索增强陈述）：
尊敬的审判长，经检索相关法律法规和司法案例，我们认为原告主张公司违法、要求支付剩余两个月经济补偿金的请求应予以支持。
首先，根据《中华人民共和国劳动合同法》第四十七条明确规定：“经济补偿按劳动者在本单位工作的年限，每满一年支付一个月工资的标准支付；六个月以上不满一年的，按一年计算；不满六个月的，支付半个月工资的经济补偿。”
其次，参考最高人民法院公布的典型案例，如2020年北京市第三中级人民法院审理的编号为（2020）京03民终XXXX号案件中，法院明确裁定：
“劳动者工作年限满三年，应获得三个月工资的经济补偿；公司支付不足的部分，劳动者有权继续主张。”
本案原告张某工作年限为三年，公司只支付了一个月经济补偿金，不符合法律规定。被告公司应依法向原告补足剩余两个月工资的经济补偿金。
因此，我们建议公司主动履行剩余经济补偿义务，避免进一步的法律后果，并请求法院依法公正判决。

3.3.5  多模态证据输入
在真实的庭审调查中，证人提供的证据往往是种类多样的，如欠条的文本模态，录音笔的声音模态，监控录像的视觉模态。这些证据中蕴含着丰富的案件相关的判决信息，需要对其具有细粒度的理解才能够作出正确的判决。对于普遍的智能体系统，其庭审过程中仅有单一模态的输入，交互与输出，无法合理的利用多模态的信息进行判决。由于法律微调模型通常不具备多模态的识别能力，我们选择使用多模态通用模型，如gemini-2.0，gpt-4o等进行多模态信息的理解，将多模态的证据转化为文本模态以便于智能体进行交互。在实际使用中，我们通过调用多模态模型API的方式处理前端传入的多模态信息并输出为有结构的文本信息，用于证人阵营的智能体在法庭调查环节进行证据陈述。
