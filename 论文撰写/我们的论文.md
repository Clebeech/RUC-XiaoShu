3.3 研究方法

本研究以高校教务场景中的“政策分散、更新频繁、咨询量大、答复标准难统一”等问题为出发点，围绕“智慧教务助手建设——基于文档解析的RAG引擎技术研究”展开。研究思路与系统海报保持一致，整体分为“数据工程”“RAG引擎与幻觉抑制”“系统落地与交互”三部分推进，形成从数据治理、知识检索到产品闭环的完整方法体系。与通用问答系统不同，教务场景强调政策依据、适用对象与时间边界，若仅依赖大语言模型自身记忆进行回答，容易出现对象错配、事实偏差和引用缺失等问题。因此，本文采用“文档解析 + 检索增强生成 + 多模态输入 + 人工反馈闭环”的技术路线，将教务制度文本转化为结构化知识，再通过检索增强生成实现“有依据、可追溯、可更新”的智能答疑。

3.3.1 数据工程方法

数据工程是本研究的基础环节。结合海报中的研究链路，系统首先围绕本科教学与学籍管理场景开展数据采集，收集《本科生培养方案》《学籍管理办法》等核心政策文件，以及与课程管理、毕业要求、考试安排、学分认定相关的辅助规范材料，形成教务专用语料来源。同时，结合师生咨询记录、访谈内容与邮件问答，整理高频问题样本，用于补充真实使用场景中的表达方式与需求重点。通过这一过程，研究对象不再局限于静态政策文本，而是同时覆盖“制度知识”和“问题表达”两个维度，为后续检索与生成奠定基础。

在文档处理阶段，系统支持 TXT、Markdown、PDF、DOCX 等常见格式。针对纯文本文件，系统直接读取文本内容；针对 PDF 与 Word 文档，则分别调用对应解析组件提取页面文本和段落文本。考虑到教务资料中常包含换行噪声、表格、HTML 残留标签及图片标记等内容，系统在入库前执行统一清洗，包括规范换行、去除无效标记、将表格转换为可检索文本以及压缩多余空白。这样做的目的，是尽可能保留课程名称、适用年级、办理条件、时间节点、学分要求等关键字段，提高后续知识抽取与检索命中的稳定性。

在文本组织上，本文采用“段落优先、长度受控、重叠衔接”的切块方法。系统默认切块长度为1000字符，块间重叠长度为200字符。切块时优先保持段落完整，只有在单段过长时才继续拆分。该方法既能保留政策条文的完整语义，又能降低相邻片段截断带来的信息损失。每个文本块同时记录所属文档、块序号和起止位置，为后续引用展示与结果回溯提供结构化支持。完成切块后，系统调用嵌入模型生成向量表示，并将分块内容与向量结果一并写入知识库，形成面向教务场景的可检索索引。

3.3.2 RAG引擎与幻觉抑制方法

在RAG引擎设计上，本文强调“检索先行、证据约束、生成受控”的方法原则。用户提问后，系统先对问题进行标准化处理，例如统一年级写法、归并课程名称的不同表达形式，并提取其中的专业、课程、年级、必修要求等关键词线索。随后，系统对问题向量与知识库中文档块向量计算相似度，并叠加面向教务场景设计的启发式重排规则，如年份命中、文档标题命中、主体词匹配和高频业务词加权等，以减少“语义接近但对象不符”的误检现象。最终，从排序结果中选取Top-K片段作为生成依据，并限制单一文档的片段占比，避免回答过度依赖某一篇文档。

与一般的开放式生成不同，本文将检索结果显式注入提示词模板，要求模型优先依据知识库内容回答；若文档未明确覆盖，则需要直接说明“知识库未提供充分依据”；若答案需要综合多个片段，则应先归纳证据、再形成结论。通过这种方式，模型的生成空间被限制在可验证证据范围内，从而有效抑制幻觉。与此同时，系统在回答结果中追加参考文档信息，标注文档名称及相关度，使用户能够清楚看到答案来源。海报中强调的“句句有出处”，在本研究中具体体现为“检索结果参与生成、生成结果附带引用、引用结果支持回溯”的三层机制。

此外，本文还对输入场景进行了扩展。对于图片问题，系统先利用多模态模型读取截图中的课程名称、教师、学分、班级、年级等信息，再将“用户问题 + 图片识别结果”共同送入检索链路，使截图内容能够与本地政策文档进行联动验证。对于语音输入，系统先完成语音转写，再进入常规问答流程。多模态输入并没有改变RAG的核心逻辑，而是通过“先转文本、再检索、后生成”的统一流程，扩大智慧教务助手的实际可用范围。

3.3.3 系统落地与交互方法

为验证研究方法的可行性，本文进一步将上述技术路线落地为可运行系统。前端采用 React 与 shadcn/ui 搭建交互界面，支持用户登录、知识库切换、文档上传、文本问答、图片提问、语音转写和历史会话查看；后端采用 FastAPI 提供认证、知识库管理、文档解析、向量检索、问答生成与数据持久化能力。系统在数据库中维护用户、知识库、文档、文档块、聊天记录和反馈工单等核心实体，使研究不仅停留在算法实验层面，而是形成一个面向真实场景运行的教务助手原型。

在交互设计上，本文强调“可理解”与“可迭代”两个目标。一方面，系统通过参考文档展示、知识库分类和会话保存机制，提高回答过程的透明度与连续性；另一方面，系统引入Human-in-the-loop闭环，将用户对答案的满意度反馈与后台运营看板结合起来，对检索未命中、答案不准确、多模态识别偏差、引用不充分和流程解释不清晰等问题进行分类管理。对于低质量回答，系统可进一步推动人工复核、知识补充和索引更新。由此，智慧教务助手不再是一次性静态部署的问答模型，而是形成“知识入库—检索生成—用户反馈—人工修正—再次更新”的持续优化机制。

综上，本文的研究方法与海报展示的技术路线保持一致：以前端真实交互需求反推数据工程建设，以文档解析和RAG引擎作为核心技术支撑，再通过多模态输入与人工闭环提升系统可用性和可靠性。该方法既发挥了大语言模型在自然语言理解与生成方面的优势，又通过知识库检索、证据约束和反馈更新机制弥补了其在专业事实场景中的不足，为智慧教务助手的建设提供了具有工程落地性的实现路径。
