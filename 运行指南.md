# RAG项目运行指南

## 项目概述

这是一个基于MetaGPT导出的RAG（检索增强生成）聊天应用，使用以下技术栈：
- **前端框架**: React 19 + TypeScript + Vite
- **UI组件库**: shadcn-ui + Tailwind CSS
- **AI服务**: 通义千问（Qwen）API
- **向量存储**: 本地内存存储（简化版嵌入向量）

## 快速开始

### 1. 安装依赖

项目使用 `pnpm` 作为包管理器（版本 8.10.0）。

```bash
cd /Users/tigerxu/Desktop/rag_proj/workspace/shadcn-ui
pnpm install
```

如果没有安装 pnpm，可以使用以下命令安装：
```bash
npm install -g pnpm@8.10.0
```

或者使用 npm/yarn 安装依赖（不推荐，可能版本不匹配）：
```bash
npm install
# 或
yarn install
```

### 2. 配置API密钥（可选）

当前代码中API密钥已硬编码在 `src/lib/qwen-api.ts` 文件中：

```typescript
this.api_key = api_key || 'sk-c730dd545e724682a71a897155fe58e9';
```

如果需要使用自己的API密钥，有两种方式：

**方式1：修改代码**
编辑 `src/lib/qwen-api.ts`，替换默认的API密钥。

**方式2：通过环境变量（需要修改代码支持）**
创建 `.env` 文件：
```bash
VITE_QWEN_API_KEY=your_api_key_here
```

### 3. 启动开发服务器

```bash
pnpm run dev
```

项目将在 `http://localhost:5173` 启动（Vite默认端口）。

### 4. 构建生产版本

```bash
pnpm run build
```

构建产物将输出到 `dist/` 目录。

### 5. 预览生产构建

```bash
pnpm run preview
```

## 核心功能

### 1. 文档上传与解析
- 支持上传文档到知识库
- 自动解析文档并分块
- 生成嵌入向量用于检索

### 2. RAG查询
- 基于向量相似度检索相关文档
- 将检索结果作为上下文发送给AI
- 生成基于文档的准确回答

### 3. 聊天功能
- 多轮对话支持
- 聊天历史保存（localStorage）
- 多知识库切换
- 模型选择

## 常见问题

### 1. CORS错误

如果遇到CORS（跨域）错误，这是因为浏览器阻止了直接访问API。解决方案：

**开发环境**：
- 使用浏览器扩展禁用CORS检查
- 或配置Vite代理（推荐）

在 `vite.config.ts` 中添加代理配置：
```typescript
export default defineConfig({
  // ... 其他配置
  server: {
    proxy: {
      '/api': {
        target: 'https://dashscope.aliyuncs.com',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
});
```

**生产环境**：
- 使用后端代理服务器
- 或部署到服务端处理API调用

### 2. API密钥无效

如果API调用失败，检查：
- API密钥是否正确
- API密钥是否过期
- 账户是否有足够的额度

### 3. 依赖安装失败

如果 `pnpm install` 失败：
- 检查Node.js版本（建议 18+）
- 清除缓存：`pnpm store prune`
- 删除 `node_modules` 和 `pnpm-lock.yaml` 后重新安装

### 4. 端口被占用

如果5173端口被占用，Vite会自动使用下一个可用端口，或手动指定：
```bash
pnpm run dev -- --port 3000
```

## 技术说明

### 向量存储
当前使用简化的本地向量存储，基于词频和哈希函数生成嵌入向量。生产环境建议：
- 使用专业的嵌入模型（如OpenAI embeddings、sentence-transformers）
- 使用向量数据库（如Pinecone、Weaviate、Milvus）

### 文档解析
当前支持基本的文本解析，可以扩展支持：
- PDF文档
- Word文档
- Markdown
- 其他格式

## 开发建议

1. **环境变量管理**：建议将敏感信息（如API密钥）移到环境变量
2. **错误处理**：完善错误处理和用户提示
3. **性能优化**：大量文档时考虑使用Web Worker处理向量计算
4. **持久化存储**：考虑使用IndexedDB替代localStorage存储向量数据
